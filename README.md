# Voice Support System (Phase 4)

## Project Objective

Build a fully local, topic-scoped voice customer support system where:

- users speak naturally in the browser,
- speech is transcribed by local Whisper,
- responses are generated by a local Ollama-backed chat service restricted to the selected support topic,
- and replies are spoken back using local Piper TTS.

The system is structured as a single repository with separate service folders so each part can run independently and be reused by other projects through HTTP APIs.

## Services and Ports

| Folder | Port | Responsibility |
|--------|------|----------------|
| [services/web](services/web) | 3000 | Next.js UI |
| [services/ollama](services/ollama) | 4000 | Chat API + topic guard + knowledge-base integration |
| [services/speech](services/speech) | 8000 | STT (faster-whisper) + TTS (Piper) |
| Ollama daemon | 11434 | Model runtime (`ollama serve`) |

## End-to-End Flow

```text
Mic input
  -> services/speech /stt
  -> transcribed text
  -> services/ollama /chat
  -> Ollama daemon response
  -> services/speech /tts
  -> audio reply
```

## Quick Start

1. Install dependencies at repo root:

   ```bash
   npm install
   ```

2. Start Ollama daemon and pull model (from repo root):

   ```bash
   npm run ollama:serve
   npm run ollama:pull
   ```

3. Start the Ollama chat API:

   ```bash
   npm run dev:ollama
   ```

4. Start the speech service:

   ```bash
   npm run dev:speech
   ```

5. Start the web app:

   ```bash
   npm run dev:web
   ```

6. Open `http://localhost:3000`.

### Start all services (separate terminal windows)

| Platform | Command |
|----------|---------|
| **macOS Terminal** | `npm run start:all:mac` |
| **Windows Terminal** | `npm run start:all:win` |
| **Windows CMD** | `npm run start:all:win:cmd` |

Each opens 4 windows/tabs: Ollama daemon, Ollama API, speech service, web UI.  
On Windows you can also double-click `scripts/start-all-terminals.cmd` (same as CMD).

You can also run web + ollama API in one terminal with `npm run dev` (still start Ollama daemon and speech separately, or use a start script above).

## Environment Files

| Service | File | Key variables |
|---------|------|---------------|
| Web | `services/web/.env` | `NEXT_PUBLIC_OLLAMA_SERVICE_URL`, `NEXT_PUBLIC_SPEECH_URL` |
| Ollama API | `services/ollama/.env` | `OLLAMA_MODEL`, `OLLAMA_BASE_URL`, `TOPIC_STRICT_MODE`, `KNOWLEDGE_BASE_DIR` |
| Speech | `services/speech/.env` | `WHISPER_MODEL`, `PIPER_VOICE`, `CORS_ORIGINS` |

Copy each `.env.example` to `.env` as needed.

## Knowledge Base

Topic content is stored under:

`services/ollama/knowledge-base/{topicId}/`

Edit these markdown files to update what the assistant can answer for each support line.

## Testing

Run Playwright end-to-end tests:

```bash
npm run test:e2e
```

Playwright starts `services/web` on port `3099` and mocks calls to ollama and speech APIs.

Optional stop-reply checks (require running web on `:3000` and ollama API on `:4000` for the Node script):

```bash
npm run test:stop-reply
npm run test:stop-reply:browser
```

Scripts live in `e2e/test-stop-reply.mjs` and `e2e/test-stop-reply-browser.mjs`.

## Troubleshooting

See [TROUBLESHOOTING.md](TROUBLESHOOTING.md).

| Problem | Quick fix |
|---------|-----------|
| Ollama daemon unreachable | Run `npm run ollama:serve` |
| Chat API unavailable | Run `npm run dev:ollama` |
| Speech API unavailable | Run `npm run dev:speech` |
| Browser CORS errors | Ensure `CORS_ORIGINS` includes `http://localhost:3000` |
